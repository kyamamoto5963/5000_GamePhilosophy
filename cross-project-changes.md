# 横展開チェンジレジストリ（正本）

最終更新: 2026-06-18

複数タイトルに**同じ修正を波及させたい**ときの台帳。ここ（`5000_GamePhilosohy`）が**正本**。
「philosophy や共通方針を変えたら、各タイトルにも反映が要る」類の変更を、**ID 付き**で1件1行管理する。

## 仕組み

> 大前提: **他プロジェクトを別セッションから直接いじらない**。横展開は「ここに起票」までにとどめ、
> 反映は各プロジェクトの責任で（デグレ防止）。→ [philosophy.md](philosophy.md)「作業スコープ」。

1. **変更は必ず ID を振ってここに起票する**。1 変更 = 1 エントリ。「何を・なぜ・どう直すか」「対象タイトル」「正本側の反映先」を書く。
2. **各プロジェクトは自分の完了リストを持つ**（各リポジトリの `docs/cross-project-changes-status.md`）。状態は **未対応 / 対応中 / 完了 / 対象外(N/A)**。
3. 各プロジェクトで作業を始めるとき、この台帳の「未完了で自分が対象の CPC」を確認 → 反映したら status と下の表を更新。

## 採番ルール

- 形式: **`CPC-NNNN`**（Cross-Project Change、4桁ゼロ詰め連番）。
- 次番号は下の一覧の最大 +1。欠番を埋め直さない。一度発行した ID の意味は変えない。
