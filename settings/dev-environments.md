# 開発環境（2拠点：会社／自宅）

開発作業は会社と自宅の2つの場所で行う。

## 作業開始時の合図
ユーザーが作業前に、どちらの場所かを伝える。

- 「会社で作業します」
- 「自宅で作業します」

## 拠点で異なる値の扱い
ファイルパスなど、場所によって異なる箇所がある。これらはベタ書きせず、
開発開始時に切り替えられる形にしておく。

切り替え手段:

- ソース内のパスや数値 … コメントアウトで2拠点分を並記し、使う側を有効化
- ビルド分岐したいもの … 条件コンパイル（例: `OFFICE` / `HOME` のフラグ）で分岐

## 拠点差がある値の一覧
場所差が見つかるたびに、ここへ追記していく。

| 項目 | 会社 | 自宅 | 切り替え手段 | 備考 |
|---|---|---|---|---|
| （例）プロジェクトルート | | | コメントアウト | |
