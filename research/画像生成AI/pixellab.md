# PixelLab 調査メモ

最終更新: 2026-06-25
カテゴリ: 画像生成AI サービス調査
公式: https://www.pixellab.ai/

## 何者か

**ピクセルアート特化**の生成 AI サービス。汎用画像 AI（Midjourney 等）と違い、
出力が最初から「ドット絵のゲームアセット」になるのが売り。キャッチコピーは
「Create unique pixel art assets for your game 10x faster with AI」。
インディーのゲーム開発者を主なターゲットにしている。Windows / Mac / Linux 対応。

## できること

| 種類 | 内容 |
|---|---|
| キャラ生成 | テキスト → キャラスプライト。4方向 / 8方向の回転ビュー、アイソメ対応 |
| アニメーション | ワンクリック生成、スケルトン（ボーン）制御、テキストプロンプト指定。歩く/走る/攻撃など。4〜16フレーム、フレーム補間あり |
| タイルセット / 背景 | テキストからシーン生成、テクスチャ・タイルセット作成。トップダウン / 横スクロール / アイソメ |
| UI部品 | メニュー・ボタン・体力バー等 |
| 編集系 | インペイント（スタイル合わせの部分修正）、背景除去、スタイル参照画像での一貫生成、衣装転送、AIアップスケール |

スタイル一貫性（reference image でアセット間の絵柄を揃える）と、ボーンベースの
アニメ制御が、ゲーム量産用途で効く部分。

## Creator と Characters の使い分け（実地メモ）

Web ツールには似た入口が複数ある。特に紛らわしいのが **Creator** と **Characters**。

| | Creator（Simple Web Creator） | Characters（Instant Character Creator） |
|---|---|---|
| 正体 | 軽量の汎用画像ジェネレータ | キャラ専用のスプライト生成パイプライン |
| 出力 | **1枚絵**（静止スプライト1体） | **4/8方向 ＋ アニメ付きのスプライトシート一式** |
| 入力 | テキスト（自由度高い） | テキスト中心、方向・アニメ前提の制約あり |
| モデル | PixFlux / BitForge | 方向一貫性・アニメ最適化の専用処理 |
| 向く用途 | 良い絵柄を1体当てる | ゲーム即投入の歩走攻撃＋全方向 |

**所感（2026-06-25 実際に触って）**: Creator はキャラを作るといい感じのが一体できる。
一方 Characters はまだ当たりが出ない。これは仕様上ほぼ必然で、Creator は「1枚の良い絵」を
出すだけなので自由度が高くガチャ運も良いが、Characters は「同じキャラを全方向＋アニメで
破綻なく揃える」という遥かに厳しい制約を同時に満たすため、1フレームの質を Creator ほど
詰めにくい。

**使い分けの方針**:
1. Creator で当てた良いキャラを**「種」**にする。Characters やスタイル参照（reference image）に
   その1枚を食わせ、絵柄を固定したまま方向・アニメへ展開する。ゼロからテキストだけで
   Characters を回すより当たりやすいはず。
2. それでも崩れるフレームは**インペイントで部分修正**して仕上げる。

## 料金

二本立て。**Web ツール/Aseprite 拡張のサブスク**と、**API の従量課金**が別建て。

### サブスク（Web / Aseprite拡張）
| プラン | 月額 | 主な内容 |
|---|---|---|
| 無料トライアル | $0 | 高速生成40回・クレカ不要 |
| Tier2 Pixel Artisan | $24/月 | 優先キュー、画像上限 400×400px |
| Tier3 Pixel Architect | $50/月 | 最優先、同時20ジョブ、チーム共同編集 |

サブスクには月あたりの生成プールが付く。

### API（従量）
- クレジット課金。1操作あたり **$0.002〜$0.185** 程度。
- 価格は「出力サイズ・モデル（basic vs Pro）・オプション（透過背景・スタイル参照）」で変動。
- 出力サイズ目安: **16×16 〜 512×512px**。標準モデルは 200〜400px 上限、Pro ツールで 512px まで。
- モデルは Pixflux / Pixen / Bitforge など複数。
- 認証はトークン方式（アカウント設定で API トークンを取得）。
- API ドキュメント: https://api.pixellab.ai/v2/docs

## 連携手段

- **Aseprite 拡張**: ドット絵エディタ Aseprite 内から直接生成。実制作に乗せやすい。
- **API**: 手続き生成・ゲーム内のライブアセット生成・動的コンテンツ向け。
- **MCP（Vibe Coding）**: Claude Code / Cursor 等の AI コーディングアシスタントから
  IDE 内でアセット生成可能。リポジトリ `pixellab-code/pixellab-mcp`。

## 本作での使いどころ・所感

- KeyInputGame 系のような **2D ドット絵アクション**ならドンピシャ。4/8方向・歩走攻撃の
  スプライトシートをまとめて出せるのは量産で強い。
- スタイル参照でアセット間の絵柄を揃えられるので、「キャラごとにバラつく」生成 AI の
  典型的弱点を抑えられる。
- まず**無料40回**で絵柄の質と方向性が本作の雰囲気に合うか試す。合えば Aseprite 拡張で
  制作フローに組み込み、量産フェーズで API を検討、という順が無理がない。
- 未確認: 商用利用ライセンスの細目、生成物の権利、日本語プロンプト精度。導入前に要確認。
