# GamePhilosophy 調査メモ（research）

「測ってみた・調べてみた」の結論を貯める場所。罠・教訓を残す `knowledge/` とは役割が違い、
こちらには **性能の計測値・挙動の検証・技術選定の裏づけ**といった「判断のもとになる事実」を置く。

仕分けの目安:

- `knowledge/` … また踏むやつ。症状 → 原因 → 対処の罠・教訓。
- `research/` … 実際どうなの？ 計測・実験・比較・調査の結論。

## 索引

- [compare_damage-formula.md](compare_damage-formula.md) — ダメージ計算式の比較・選定（除算方式ベース＋本作向け補正：ランダム抑制ほか）

## 書き方ルール

- ファイル名は `<カテゴリ>_<短い説明>.md`（例: `perf_*` / `compare_*` / `spike_*`）。
- 本文の流れ: きっかけ・問い → 測ったもの → 結果 → なぜそうなるか → だからどう判断するか → 再現手順。
- 数値は条件込みで書く（端末・件数・試行回数）。推測と実測は混ぜない。
- 日付は絶対日付。ファイルを足したらこの索引にも 1 行足す。
